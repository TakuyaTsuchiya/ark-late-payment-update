# ArkLatePaymentUpdate

**アーク残債取り込みアプリケーション**

## 概要
アークからの残債データをミライル顧客システムに取り込み可能な形式に変換する自動化ツールです。
2つのCSVファイルを紐付けて、管理番号ベースの残債情報CSVを生成します。

## 主な機能

### データ処理機能
- **自動ファイル検索**: 指定パターンのCSVファイルを自動検出
- **エンコーディング自動判定**: CP932, Shift_JIS, UTF-8に対応
- **データ紐付け**: 契約番号と引継番号での自動マッチング
- **品質チェック**: 紐付け成功率の統計表示

### 入力ファイル
1. **【アーク継続中】②残債取込用CSV_*.csv**
   - アークからの残債データ
   - 主要項目: 契約番号、未収金額合計

2. **ContractList*.csv**
   - ミライル顧客システムからエクスポートしたデータ
   - 主要項目: 管理番号、引継番号

### 出力ファイル
- **mmddアーク残債.csv** (例: 0801アーク残債.csv)
- 構造: 管理番号, 管理前滞納額 (2列)
- エンコーディング: CP932

## 使用方法

### 基本的な使い方
```bash
# プロジェクトディレクトリに移動
cd ark_late_payment_update

# 依存パッケージのインストール
pip install -r requirements.txt

# 入力ファイルをプロジェクトディレクトリに配置
# - 【アーク継続中】②残債取込用CSV_YYYYMMDD.csv
# - ContractList_YYYYMMDD.csv

# 実行
python src/main.py
```

### 処理フロー
1. **ファイル検索**: パターンマッチングで入力ファイルを自動検出
2. **データ読み込み**: エンコーディング自動判定でCSV読み込み
3. **データ紐付け**: 契約番号 = 引継番号でのマッチング
4. **データ抽出**: 管理番号と未収金額合計を抽出
5. **CSV出力**: CP932エンコーディングで出力

## 技術仕様

### 動作環境
- **言語**: Python 3.8+
- **主要ライブラリ**: 
  - pandas 2.0.3: データ処理
  - chardet 5.2.0: エンコーディング検出
- **対応OS**: Windows
- **エンコーディング**: CP932（Shift_JIS）

### パフォーマンス
- **処理容量**: 10,000件程度のデータに対応
- **処理時間**: 約1,000件/10秒
- **メモリ使用量**: 約100MB（1,000件処理時）

## データマッピング

### 紐付けキー
- アーク残債CSV: 「契約番号」
- ContractList: 「引継番号」
- **この2つが同じ値として紐付け処理**

### 出力マッピング
- A列「管理番号」 ← ContractListの「管理番号」
- B列「管理前滞納額」 ← アーク残債CSVの「未収金額合計」

## トラブルシューティング

### よくある問題

**1. ファイルが見つからない**
```
FileNotFoundError: 指定パターンのファイルが見つかりません
```
→ ファイル名がパターンと一致しているか確認
→ プロジェクトディレクトリにファイルが配置されているか確認

**2. エンコーディングエラー**
```
UnicodeDecodeError: codec can't decode
```
→ ファイルがCP932またはUTF-8でエンコードされているか確認
→ chardetによる自動検出で対応済み

**3. 紐付けデータが少ない**
```
WARNING: 紐付け成功率が低い (XX%)
```
→ 契約番号と引継番号の値が一致しているか確認
→ データ形式（文字列/数値）の差異を確認
